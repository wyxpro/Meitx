# 示例：生成一个「附近跑腿配送」微信小程序

展示本 skill 在小程序模式下从需求分析到合规注入的完整走查。

## 用户需求

> 帮我做一个微信小程序：用户可以发布跑腿订单（填写取货地址、收货人姓名和电话），骑手抢单后用地图导航送达，订单完成可以评价。需要登录才能下单。

## Step 1 — 分析需求，推断合规组件

**识别功能特征：**
- 「需要登录才能下单」「收货人电话」→ 登录授权 → **M03、M05、M09**（含登录功能即触发 M05 首页登录规范、M09 账号注销）
- 「地图导航」「取货/收货地址」「附近」→ 系统能力授权（位置）→ **M03**

**全局必选：** M01、M02、M07

**合规组件清单：** M01、M02、M03、M05、M07、M09

**前置信息确认：** 无敏感信息（M04 未触发）、无个性化推荐（M06 未触发）、无未成年人特征（M08）→ 无需向用户追加提问。

## Step 2 — 注入合规组件（按用户流程顺序）

1. **M01 隐私政策弹窗** — 首次进入即弹出。「同意」/「不同意」等大按钮；不预勾选；不同意可进入仅浏览模式（可看附近订单，不能下单）。
- **登录页协议默认不勾选（M01 在登录场景的展现形态）** — 因本小程序含「注册登录」，登录页（手机号/第三方登录）须在登录按钮上方/下方提供「我已阅读并同意《用户服务协议》与《隐私政策》」勾选项；复选框**默认未勾选**，由用户主动勾选；未勾选点击登录须给出提示（轻提示/抖动）引导先勾选，不得代为勾选或跳过；禁止"登录即代表同意"等默示同意话术。
2. **M03 授权弹窗（位置，系统能力授权）** — 用户点「查看附近订单 / 导航」时才申请位置授权，非进入即申请；调起微信原生授权弹窗，附用途说明。
3. **M03 授权弹窗（登录，用户隐私保护授权）** — 用户点「下单」时才触发手机号登录；首页不强制登录；提供「暂不登录 / 返回」出口。处理手机号等个人信息**优先采用微信官方隐私弹窗**，无需自行设计弹窗。
4. **M05 首页登录规范** — 本小程序面向公众，首页不得打开即要求登录；先展示附近订单列表，用户点「下单」才触发登录；登录页提供「暂不登录 / 返回」明确出口；登录页协议勾选默认未勾选（见 M01）。
5. **M02 隐私政策查看入口** — 「我的-设置」放置隐私政策入口，路径≤4层。
6. **M07 双清单查看入口** — 「我的-设置」放置已收集个人信息清单（6 要素，登录后访问、支持近7天/30天/1年切换、敏感信息脱敏）+ 第三方信息共享清单（7 要素，含地图服务商）查看入口。
7. **M09 账号注销流程** — 因含登录功能而注入。「我的-设置-账号管理-账号注销」，5 次操作内可达；注销前展示数据清除说明；注销/取消按钮等视觉权重。

> 拒绝授权的降级处理：拒绝位置仍可手动输入地址下单，拒绝登录仍可浏览订单列表；不退出、不循环弹窗。微信侧若距用户上次拒绝不足 10 秒，不再弹出隐私弹窗。

## Step 2.4 — 合规实现指引（内部 metadata 示例）

```
[compliance] M03 授权弹窗（位置）
- 法规依据：T/TAF 180.1 §5.5；微信规范"未体验功能前要求授权"
- 触发条件：用户点击"查看附近订单"或"导航"时
- 交互要求：功能触发时调起微信原生授权弹窗，附用途说明"用于展示附近订单和配送导航"；不在进入即申请
- 状态流转：未授权 → 已授权；可在微信设置撤回
- 开发注意事项：严禁启动即申请；拒绝后保留手动输入地址路径，48h 内不就该授权频繁弹窗
```

```
[compliance] M03 授权弹窗（登录，个人信息）
- 法规依据：微信《收集用户隐私行为规范与修改指引》
- 触发条件：用户点击"下单"需获取手机号时
- 交互要求：优先采用微信官方隐私弹窗（耦合样式或直接弹窗样式）；用户同意后方可使用；拒绝保留降级
- 状态流转：未同意 → 已同意；可在微信侧撤回
- 开发注意事项：须先在小程序管理后台声明所处理的用户信息；距上次拒绝不足 10 秒不再弹出
```

## Step 3 — 反面模式自检（节选）

- [x] M01 同意/不同意等大、未预勾选、拒绝后可仅浏览
- [x] M03 位置/登录均在功能触发时申请，未在启动时一次性索取
- [x] M03 位置用微信原生授权弹窗；登录获取手机号优先用微信官方隐私弹窗，未自行设计授权弹窗
- [x] M05 面向公众首页不强制登录，提供"暂不登录/返回"出口
- [x] 拒绝位置仍可手填地址，拒绝登录仍可浏览
- [x] 设置页含隐私政策、双清单（M07）入口，路径≤4层
- [x] M09 注销入口在「设置-账号管理」下，5 次操作内可达，注销/取消按钮等权重

## Step 4 — 输出

生成所有业务页面（首页订单列表、发布订单、订单详情、我的）+ 上述合规组件，组件在代码中以 `// [compliance] M0X 名称` 标注。

## 对比：哪些被本 skill 拦下了

| 常见违规做法 | 本 skill 的修正 |
|-------------|----------------|
| 进入小程序即弹登录、不登录无法浏览 | 首页先展示订单列表，下单时才触发登录，并提供"暂不登录"出口（M05） |
| 进入页面即一次性申请位置+登录 | 拆为各自功能触发时申请（M03） |
| 隐私弹窗预勾选 / 只有"同意"按钮 | 明示同意，等大同意/不同意按钮，不预勾选（M01） |
| 自行设计个人信息授权弹窗、用普通按钮 | 处理个人信息优先用微信官方隐私弹窗（M03） |
| 设置里找不到双清单 | 「我的-设置」提供已收集信息 + 第三方共享双清单入口（M07） |
