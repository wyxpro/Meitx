# 示例：生成一个「运动健身记录」App

展示本 skill 在 App 模式下从需求分析到合规注入的完整走查。

## 用户需求

> 帮我做个 App：用户注册登录后可以记录跑步、骑行，用 GPS 画运动轨迹，拍照打卡分享到微信，开通会员可解锁高级数据分析。Android 和 iOS 双端。

## Step 1 — 分析需求，推断合规组件

**识别功能特征：**
- 「注册登录」→ 注册登录 → **C04**
- 「GPS 画运动轨迹」「跑步/骑行」→ 位置 → **C06**
- 「拍照打卡」→ 摄像头 → **C06**（触发 C06 同时注入 **C05** 应用权限说明入口）
- 「开通会员解锁」→ 自动续费/付费服务 → **C10**

**全局必选：** C01、C02、C03

**合规组件清单：** C01、C02、C03、C04、C05、C06、C10

**前置信息确认（Step 1.2）：**
- C06 被触发 → 已知双端（Android + iOS）→ 安卓系统弹窗前用蒙层/顶栏浮窗告知目的且每次申请均展示；iOS 复用系统弹窗自带的目的说明
- C10 被触发 → 须确认会员是否多档？→ 假设为「月卡/年卡」两档 → 须展示权益对比表
- C07（个性化推荐）未触发；C09（未成年人）需求未涉及，触发模糊 → 主动询问「是否面向未成年人 / 允许 18 岁以下注册？」假设答否，不注入 C09。

## Step 2 — 注入合规组件（按用户流程顺序）

1. **C01 隐私政策同意弹窗** — App 启动第一步弹出。「同意」/「不同意」等大按钮；不预勾选；不同意可挽留一次，二次拒绝后退出或进入仅浏览模式。
   - **登录页协议默认不勾选（C01 在登录场景的展现形态）** — 因本 App 含「注册登录」，登录页（手机号/第三方登录）须在登录按钮上方/下方提供「我已阅读并同意《用户服务协议》与《隐私政策》」勾选项；复选框**默认未勾选**，由用户主动勾选；未勾选点击登录须给出提示（轻提示/抖动）引导先勾选，不得代为勾选或跳过；禁止"登录即代表同意"等默示同意话术。
2. **C06 权限申请弹窗（位置）** — 用户点「开始运动」时才申请；用途说明「用于记录运动轨迹」；安卓在系统弹窗前以蒙层/顶栏浮窗告知，iOS 复用系统弹窗话术；拒绝后可手动记录里程，不退出。
3. **C06 权限申请弹窗（摄像头）** — 用户点「拍照打卡」时才申请；独立弹窗；拒绝后仍可发文字打卡。
4. **C10 自动续费服务告知** — 会员开通页：自动续费为独立勾选框且默认不勾选（禁止“点击开通即同意自动续费”的默示同意，须有勾选框）；展示月卡/年卡权益对比表；续费前 5 日短信/推送提醒；提供随时退订入口。
5. **C02 隐私政策查看入口** — 「我的-设置」列表行，路径 ≤4 次点击。
6. **C03 双清单** — 「我的-设置」放置已收集个人信息清单（6 要素，登录后访问、支持近7天/30天/1年切换、敏感信息脱敏）+ 第三方信息共享清单（7 要素，含微信分享 SDK）。
7. **C05 应用权限说明查看入口** — 「我的-设置」权限管理入口，展示全部已申请权限及用途，与 manifest 声明一致。
8. **C04 账号注销流程** — 因含注册登录而注入。「我的-设置-账号管理-账号注销」，5 次操作内可达；注销前展示数据清除范围 + 已开通会员处理说明 + 法定保留期限；身份验证；注销/取消按钮等权重。

## Step 2.x — 合规实现指引（内部 metadata 示例）

```
[compliance] C01 登录页协议默认不勾选
- 法规依据：164号文第1条；App违法违规收集使用个人信息行为认定方法
- 触发条件：用户进入登录页（手机号一键登录 / 第三方登录）
- 交互要求：登录按钮上方/下方提供「我已阅读并同意《用户服务协议》与《隐私政策》」勾选项，协议名称用书名号+品牌色标识可点击跳转全文；复选框默认未勾选；未勾选点击登录给出轻提示/抖动引导先勾选
- 状态流转：未勾选（默认）→ 用户主动勾选 → 可登录
- 开发注意事项：禁止预勾选或代为勾选；禁止"登录即代表同意"等默示同意话术；不得跳过勾选直接登录
```

```
[compliance] C06 权限申请弹窗（位置）
- 法规依据：191号文第2条
- 触发条件：用户点击"开始运动"时
- 交互要求：系统弹窗前以蒙层/顶栏浮窗告知目的"用于记录运动轨迹"；安卓每次申请均展示，iOS 复用系统弹窗自带话术；不叠加自定义"允许/拒绝"弹窗
- 状态流转：未授权 → 已授权；可在系统设置撤回
- 开发注意事项：严禁启动时一次性申请；拒绝后 48h 内重复弹窗不超过 1 次；拒绝后保留手动记录里程路径
```

```
[compliance] C10 自动续费服务告知
- 法规依据：工信部服务规范
- 触发条件：用户进入会员开通页
- 交互要求：自动续费默认不勾选；展示月卡/年卡权益对比表；提供随时退订入口
- 状态流转：未开通 → 已开通（手动续费 / 自动续费）
- 开发注意事项：续费前 5 日短信/推送提醒；禁止默认勾选或强制捆绑自动续费
```

## Step 3 — 反面模式自检（节选）

- [x] C01 同意/不同意等大、未预勾选、二次拒绝后不再强制弹窗
- [x] C06 位置/摄像头均在功能触发时申请，未在启动时一次性索取；安卓系统弹窗前已加蒙层/顶栏浮窗且每次展示，iOS 复用系统弹窗话术
- [x] C06 拒绝位置仍可手动记录里程，拒绝摄像头仍可文字打卡
- [x] 设置页含隐私政策、双清单（C03）、权限说明（C05）入口，路径 ≤4 次点击
- [x] C04 注销入口在「设置-账号管理」下，5 次操作内可达，注销/取消按钮等权重
- [x] C10 自动续费默认未勾选，多档已展示权益对比表
- [x] 登录页协议勾选框默认未勾选、未勾选点击登录给出提示、无"登录即同意"默示话术（C01 登录场景）

## Step 4 — 输出

生成所有业务页面（运动记录、轨迹地图、打卡分享、会员中心、我的）+ 上述合规组件，组件在代码中以 `// [compliance] C0X 名称` 标注。

## 对比：哪些被本 skill 拦下了

| 常见违规做法 | 本 skill 的修正 |
|-------------|----------------|
| 启动时一次性弹出位置+摄像头+通知权限 | 拆为各自功能触发时申请（C06） |
| Android 直接索权、无目的告知 | 安卓系统弹窗前用蒙层/顶栏浮窗告知用途（每次展示），iOS 复用系统弹窗话术（C06） |
| 隐私弹窗只有"同意"或预勾选 | 等大同意/不同意按钮、不预勾选、二次拒绝不强制（C01） |
| 启动页底部"继续即表示同意隐私政策"+单个开始按钮 | 改为标准 C01 同意弹窗，等权重「同意」/「不同意」，禁止默示同意（C01） |
| 会员页默认勾选自动续费、无续费提醒 | 默认不勾选、续费前 5 日提醒、随时退订（C10） |
| 设置里找不到账号注销 | 「设置-账号管理」5 次操作内可达，含数据清除说明（C04） |
