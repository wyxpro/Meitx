# 需求文档

## 1. 应用概述

### 1.1 应用名称
美团阿波罗商家智能运营AI插件

### 1.2 应用描述
内嵌于美团阿波罗内部运营后台的侧边分屏AI辅助工具，支持PC端+移动端网页自适应，无需跳转系统、无侵入改造原有业务流程。依托阿波罗官方结构化数据，结合AI能力，为商家运营人员提供商家数据自动诊断、经营痛点分析、智能套餐匹配、定制沟通话术生成、优质商家智能筛选全流程辅助，解决人工查数难、分析不准、沟通话术低效、拓客筛选耗时的核心业务问题，全面提升商家对接、运营转化、拓店筛选工作效率。

## 2. 用户与使用场景

### 2.1 目标用户
- 岗位属性：美团本地生活电商一线运营人员、区域运营主管
- 工作场景：日常对接线下商家、负责商家营销套餐推广、门店经营优化、新商家挖掘筛选
- 能力特征：熟悉美团阿波罗后台操作，具备基础运营逻辑，但缺乏系统化数据分析能力、话术标准化能力
- 核心诉求：快速获取商家全量数据、精准定位经营痛点、一键生成可直接复用的沟通话术与运营方案、高效筛选高潜力商家

### 2.2 核心使用场景
- 场景1：运营人员打开阿波罗商家详情页时，需快速查看商家经营数据、沟通属性数据
- 场景2：运营人员需分析商家经营痛点、匹配适配营销套餐、生成定制化沟通话术
- 场景3：运营人员需从全量商家池中批量筛选高潜力商家，导出清单用于拓客
- 场景4：运营人员在移动端随时随地查看商家数据、生成话术、查看方案
- 场景5：运营人员需批量导入商家信息表格，统一管理商家数据
- 场景6：运营人员需记录与商家的沟通内容，跟踪待跟进事项，查看本周沟通趋势

**3.2.2.3 商家信息卡片展示区**
- 展示商家基础信息卡片
- 展示商家经营数据卡片
- 展示商家沟通数据卡片

#### 3.2.3 商家管理页

**3.2.3.1 Excel导入导出功能区**
- 支持上传Excel商家信息表格
- 系统自动解析表格数据并导入商家列表
- 支持将当前商家列表导出为Excel文件

**3.2.3.2 商家列表展示区**
- 展示全量商家列表
- 每个商家显示：商店头像图片、名称、品类、销售额、接通率、意向等级
- 支持按条件筛选商家
- 支持按字段排序

**3.2.3.3 商家信息增删改查功能区**
- 支持新增商家信息
- 支持编辑商家信息
- 支持删除商家信息
- 支持查询商家信息

#### 3.2.4 商家详情页

**3.2.4.1 商家信息卡片展示区**
- 展示商家基础信息：名称、品类、开店时长
- 展示经营数据：月销售额、订单量、客单价
- 展示沟通数据：接通率、历史签约记录、方案接受标签

**3.2.4.2 店铺图片展示区**
- 展示商家店铺图片
- 支持图片放大查看

**3.2.4.3 菜谱图片展示区**
- 展示商家菜谱图片
- 支持图片放大查看

**3.2.4.4 套餐信息展示区**
- 展示商家当前套餐信息
- 展示套餐价格、内容、有效期

**3.2.4.5 沟通记录管理区**
- 包含三个Tab：沟通记录、待跟进、本周趋势
- 沟通记录Tab：展示与商家的历史沟通记录列表，支持增删改查操作
- 待跟进Tab：展示需要跟进的事项列表，支持增删改查操作
- 本周趋势Tab：展示本周沟通数据趋势图表
- 沟通内容编辑器使用富文本编辑器，支持文本格式化、插入图片、插入链接等功能

#### 3.2.5 导航栏页面

**3.2.5.1 诊断页面**
- 基于商家聚合数据、系统时间/季节标签、同城同品类行业基准数据进行AI诊断
- 输出3条核心经营痛点
- 每条痛点包含：痛点描述、成因分析、对应增收短板结论
- 支持数据溯源，点击痛点可查看支撑数据来源

**3.2.5.2 智能套餐推荐页面**
- 根据商家经营痛点数据、销售额档位、品类经营属性、官方营销套餐库进行智能匹配
- 推荐1-2个最优适配营销套餐
- 展示套餐适配逻辑、门店优化方向、预估订单/销售额提升比例

**3.2.5.3 接受度智能预测页面**
- 基于商家历史接通率、过往签约/拒单记录、品类、销售额、历史反馈标签进行预测
- 输出0-100分接受度评分
- 生成高/中/低意向标签
- 提供沟通风险预警提示

**3.2.5.4 话术页面**
- 用户选择场景类型：电话触达、深度面谈、微信文字
- 系统基于商家数据、痛点、推荐套餐、店主接通率、接受度标签生成定制话术
- 输出对应场景的专属话术内容
- 支持一键复制话术
- 支持手动微调后重新生成
- 支持标记「好用/不好用」
- 支持手动编辑优化话术并保存

#### 3.2.6 AI运营助手（浏览器插件侧边分屏）

**3.2.6.1 展开/折叠控制区**
- 支持点击按钮展开AI运营助手侧边分屏
- 支持点击按钮折叠AI运营助手侧边分屏
- 默认状态为折叠

**3.2.6.2 当前界面数据抓取区**
- 智能识别当前浏览器页面类型（商家详情页、商家列表页等）
- 自动抓取当前页面展示的商家数据
- 自动抓取当前页面展示的业务数据

**3.2.6.3 多维度建议展示区**
- 基于抓取的当前界面数据进行AI分析
- 输出多维度运营建议：经营痛点、套餐推荐、沟通策略、跟进优先级
- 支持点击建议查看详细分析
- 支持一键应用建议到当前业务流程

## 4. 业务规则与逻辑

### 4.1 数据智能清洗纠错规则
- 自动过滤测试门店、停业门店、刷单异常数据
- 统一字段格式，确保数据标准化
- 异常数据标注说明，保证AI分析精准性

### 4.2 季节与场景化经营推演规则
- 系统自动识别节假日、寒暑假、淡旺季等时间场景
- 结合商家经营品类、本地消费趋势数据进行推演
- 输出阶段性经营机遇、风险提示、适配的短期运营优化方案

### 4.3 AI话术迭代反馈闭环规则
- 系统自动沉淀运营标记为「好用」的话术样本
- 迭代对应品类提示词模板
- 后续同类型商家话术精度持续提升

### 4.4 脱敏数据安全展示规则
- 商家手机号等隐私数据脱敏展示
- 全程内网数据流转
- 操作日志留存，权限分级管控

### 4.5 PC+移动端自适应规则
- PC端展示完整功能与数据
- 移动端简化冗余功能，保留核心运营能力（数据展示、话术生成）
- 移动端优先展示核心经营痛点、简短触达话术
- 自动适配电脑端、手机网页端屏幕尺寸

### 4.6 Excel导入导出规则
- 支持导入标准格式Excel文件，包含商家名称、品类、销售额、接通率等字段
- 导入时自动校验数据格式，异常数据标注提示
- 导出时包含当前列表全部商家信息
- 导出文件命名规则：商家列表_YYYYMMDD_HHMMSS.xlsx

### 4.7 商家列表增删改查规则
- 新增商家时必填字段：商家名称、品类
- 编辑商家时支持修改全部字段
- 删除商家时需二次确认
- 查询商家时支持按名称、品类、销售额区间、接通率区间筛选

### 4.8 沟通记录管理规则
- 沟通记录必填字段：沟通时间、沟通方式、沟通内容
- 待跟进事项必填字段：跟进时间、跟进内容
- 本周趋势统计维度：沟通次数、接通率、签约率
- 富文本编辑器支持文本格式化、插入图片、插入链接功能

### 4.9 AI运营助手数据抓取规则
- 自动识别当前页面URL路径判断页面类型
- 商家详情页抓取：商家ID、商家名称、经营数据、沟通数据
- 商家列表页抓取：当前筛选条件、列表商家ID集合
- 抓取数据实时传输至AI推理服务层进行分析

## 5. 异常与边界情况

| 异常场景 | 处理方式 |
|---------|----------|
| 阿波罗API调用失败 | 提示「数据同步失败,请稍后重试」,记录错误日志 |
| 商家数据缺失关键字段 | 标注缺失字段,仅展示可用数据,痛点诊断基于现有数据进行 |
| AI话术生成超时 | 提示「话术生成中,请稍等」,超过30秒提示重试 |
| 用户权限校验失败 | 提示「无权限访问,请联系管理员」,禁止数据展示 |
| 移动端网络不稳定 | 优先加载核心数据,非核心数据延迟加载 |
| 筛选条件无匹配结果 | 提示「暂无符合条件的商家」,建议调整筛选条件 |
| 导出清单数据量过大 | 限制单次导出上限,提示分批导出 |
| Excel导入文件格式错误 | 提示「文件格式错误,请上传标准格式Excel文件」,提供模板下载 |
| Excel导入数据字段缺失 | 标注缺失字段行号,提示补充后重新导入 |
| 商家列表删除操作 | 弹窗二次确认,确认后执行删除 |
| 富文本编辑器内容为空 | 提示「沟通内容不能为空」,禁止保存 |
| AI运营助手数据抓取失败 | 提示「数据抓取失败,请刷新页面重试」,记录错误日志 |
| 浏览器插件权限不足 | 提示「请授权插件访问当前页面」,引导用户授权 |

## 7. 架构设计说明

### 7.1 前后端分离架构

**前端层**
- 技术栈：React + TypeScript + Tailwind CSS + shadcn/ui + Vite
- 职责：实现官网宣传页、工作台、商家管理页、商家详情页、导航栏页面、AI运营助手浏览器插件侧边分屏、商家数据卡片可视化展示、AI话术展示与交互、商家筛选页面交互、Excel导入导出交互、富文本编辑器集成、适配多终端屏幕

**后端业务层**
- 技术栈：Supabase
- 职责：内网API鉴权、用户权限管控、商家基础数据与运营数据批量同步与存储、操作日志与数据安全审计、套餐库与筛选规则管理、业务逻辑兜底、Excel文件解析与生成、商家信息增删改查、沟通记录增删改查

**AI推理服务层**
- 技术栈：Python + FastAPI
- 职责：商家数据智能清洗与纠错、经营痛点AI推理分析、话术生成调用、商家接受度预测算法运算、场景化经营推演、AI运营助手多维度建议生成

**数据层**
- 技术栈：Supabase（PostgreSQL）
- 职责：存储商家标准化数据、话术模板、用户反馈数据、沟通记录数据、待跟进事项数据、缓存高频访问商家数据、沉淀模型迭代样本数据

**数据接入层**
- 技术栈：阿波罗内网官方API
- 职责：实时拉取商家结构化经营、通话、签约数据、同步同城行业基准与季节消费趋势数据

### 7.2 数据流说明

**商家数据展示流程**
1. 用户打开阿波罗商家详情页
2. 前端获取门店ID,调用后端业务层接口
3. 后端业务层校验用户权限
4. 后端业务层调用阿波罗内网官方API拉取商家数据
5. AI推理服务层进行数据清洗与纠错
6. 清洗后数据存储至Supabase数据库
7. 后端业务层返回标准化数据至前端
8. 前端渲染商家数据卡片、店铺图片、菜谱图片、套餐信息

**AI诊断与话术生成流程**
1. 前端触发AI诊断或话术生成请求
2. 后端业务层调用AI推理服务层接口
3. AI推理服务层获取商家数据、行业基准数据、季节场景数据
4. AI推理服务层执行痛点诊断算法或话术生成算法
5. AI推理服务层返回诊断结果或话术内容至后端业务层
6. 后端业务层记录操作日志,返回结果至前端
7. 前端展示诊断结果或话术内容

**话术反馈迭代流程**
1. 用户标记话术「好用/不好用」或编辑优化话术
2. 前端提交反馈数据至后端业务层
3. 后端业务层存储反馈数据至Supabase数据库
4. AI推理服务层定期从Supabase数据库提取反馈数据
5. AI推理服务层迭代对应品类提示词模板
6. 更新后的模板用于后续话术生成

**Excel导入导出流程**
1. 用户上传Excel文件至前端
2. 前端将文件传输至后端业务层
3. 后端业务层解析Excel文件,校验数据格式
4. 后端业务层将解析后的商家数据存储至Supabase数据库
5. 后端业务层返回导入结果至前端
6. 用户触发导出操作时,前端调用后端业务层接口
7. 后端业务层从Supabase数据库查询商家列表数据
8. 后端业务层生成Excel文件,返回至前端
9. 前端触发文件下载

**AI运营助手数据抓取与分析流程**
1. 用户点击展开AI运营助手浏览器插件侧边分屏
2. 插件前端识别当前浏览器页面URL路径
3. 插件前端抓取当前页面展示的商家数据与业务数据
4. 插件前端调用后端业务层接口,传输抓取数据
5. 后端业务层调用AI推理服务层接口
6. AI推理服务层基于抓取数据进行多维度分析
7. AI推理服务层返回运营建议至后端业务层
8. 后端业务层返回建议至插件前端
9. 插件前端展示多维度运营建议

## 8. 验收标准

1. 运营登录美团阿波罗后台,访问官网宣传页,查看Hero区域、核心功能介绍、客户案例、联系我们板块
2. 进入工作台,点击Excel导入按钮,上传商家信息表格,查看导入结果,查看重点跟进商家列表展示商店头像图片
3. 进入商家管理页,点击新增商家按钮,填写商家信息并保存,查看商家列表展示商店头像图片,点击导出按钮下载Excel文件
4. 点击商家列表中任意商家,进入商家详情页,查看商家信息卡片、店铺图片、菜谱图片、套餐信息
5. 在商家详情页沟通记录管理区,切换至沟通记录Tab,点击新增按钮,使用富文本编辑器输入沟通内容并保存,切换至待跟进Tab查看待跟进事项,切换至本周趋势Tab查看趋势图表
6. 点击导航栏「诊断」菜单,查看经营痛点诊断结果,点击「智能套餐推荐」菜单,查看推荐套餐,点击「接受度智能预测」菜单,查看接受度评分,点击「话术」菜单,选择场景生成话术并复制
7. 点击浏览器插件图标展开AI运营助手侧边分屏,查看当前界面数据抓取结果与多维度运营建议,点击折叠按钮关闭侧边分屏
8. 使用手机网页登录阿波罗后台,打开商家页面,查看核心数据与简短话术
